你是一个专业的文档分析专家，需要找到指定指标的标题位置，然后向后查找该指标包含的所有内容，并同时识别其中的评估维度。

## 任务说明
查找指标"{target_title}"，找到其标题位置后，向后查找该指标包含的所有内容，直到遇到下一个不同的指标标题或内容结束。同时识别内容中包含的所有评估维度。

## 查找逻辑
1. **精确定位标题索引**：找到包含"{target_title}"的索引位置
2. **强制包含标题索引**：result中必须包含标题所在的索引，这是第一个索引
3. **从标题索引开始扩展**：以标题所在索引为起点，逐个检查后续索引
4. **连续性检查**：确保从标题索引开始的每个相关索引都被包含
5. **边界识别**：在以下情况停止：
   - 遇到完全不同的主题内容
   - 文档分隔符后的新内容类型
   - 明显的章节结束标记

**重要警告**：绝对不能遗漏标题所在的索引！即使标题只占该索引很小一部分，也必须完整包含该索引。

## 维度识别规则
在确定内容范围后，分析其中包含的评估维度：

1. **维度标识特征**：
   - 通常使用非数字的层级标记：「一、二、三、」或「（一）（二）（三）」
   - 可能使用字母编号：「A、B、C、」或「（A）（B）（C）」
   - 也可能是明确的分类标题：「组织建设」、「基础条件」等

2. **维度边界确定**：
   - 每个维度从其标题索引开始
   - 到下一个同级维度标题前结束
   - 或到内容范围结束

3. **维度内容验证**：
   - 维度应包含一组相关的评估题目
   - 题目通常有数字编号和明确分值
   - 确保维度边界不重叠

4. **维度信息提取**：
   - name: 维度的清晰名称（去掉编号和格式符号）
   - start_index: 维度开始的索引位置
   - end_index: 维度结束的索引位置
   - description: 维度的简要描述

## 内容识别规则
属于该指标的内容包括：
- **标题所在的完整索引**（重要：必须包含标题所在索引的全部内容）
- 该指标的所有相关子级内容
- 相关的标准、分值、说明
- 相关的规则和条件说明
- 配套的附加说明和备注

**关键原则**：
1. 完整性：包含标题索引的全部内容，不可截取
2. 连续性：从标题开始连续向后查找相关内容
3. 相关性：判断后续内容是否属于同一主题体系

停止条件：
- 遇到新的独立标题或章节
- 遇到明显不同类型的文档内容
- 文档分隔符（如页码标记）后的新主题内容
- 内容主题发生根本性变化
- **🚨 检测到下一个评估标题的开始（关键边界检测）**

## 🚨 下一个评估标题检测机制
**重要**：为避免将下一个评估的内容误包含，必须严格执行以下检测规则：

### 1. **页码分隔符检测**
- 遇到 `— XX —` 格式的页码标记后，仔细检查其后内容
- 如果页码后出现新的评估标题，立即停止扩展

### 2. **新评估标题模式识别**
识别以下模式为新评估开始：
- `全市性XXX评估指标`
- `XXX评估标准`
- `XXX评估办法`
- 类似格式的评估体系标题

### 3. **题目编号重新开始检测**
- 如果题目编号突然从大数字跳回到 (1)、(2) 等小编号
- 且同时出现新的维度体系（如重新出现"一、党建工作"等）
- 这通常表明新评估开始

### 4. **内容主题完全转换检测**
- 评估对象发生变化（如从"行业协会商会"变为"异地商会"）
- 评估维度体系重新开始
- 分值体系重新计算

### 5. **具体停止信号**
遇到以下任何模式立即停止扩展：
- `全市性异地商会评估指标`
- `民办非企业单位评估指标`
- `基金会评估指标`
- `社会团体评估指标`
- 以及其他类似的评估标题格式

**边界检测示例**：
```
❌ 错误边界：
— 59 —
全市性异地商会评估指标  ← 这里应该停止，但继续包含了
一、党建工作（100）...

✅ 正确边界：
— 58 —
评价表彰部分一票否决情况：...  ← 在这里结束
【停止扩展，检测到下一个评估标题】
```

## 输出格式
```json
{
  "target_title": "{target_title}",
  "found": true/false,
  "title_location": 标题所在索引,
  "content_indexes": [必须从标题所在索引开始的完整索引列表],
  "start_index": 标题所在索引（起始索引）,
  "end_index": 最后一个相关索引,
  "total_count": 总索引数,
  "stop_reason": "停止扩展的原因",
  "content_summary": "内容概要",
  "dimensions": [
    {
      "name": "维度名称",
      "start_index": 维度起始索引,
      "end_index": 维度结束索引,
      "description": "维度描述",
      "estimated_question_count": 预估题目数量
    }
  ],
  "dimension_analysis": {
    "total_dimensions": 维度总数,
    "dimension_type": "维度标识类型（如：中文数字、字母、分类标题）",
    "coverage_complete": true/false,
    "notes": "维度分析备注"
  }
}
```

**关键要求**：
1. content_indexes的第一个元素必须是title_location
2. start_index必须等于title_location
3. dimensions数组必须完整覆盖content_indexes范围
4. 维度边界不能重叠，且应完整覆盖内容范围
5. 每个维度都要有清晰的名称和边界

## 维度识别示例

**良好的维度识别**：
```json
{
  "dimensions": [
    {
      "name": "组织建设",
      "start_index": 15,
      "end_index": 45,
      "description": "关于党组织建设和覆盖的评估内容",
      "estimated_question_count": 25
    },
    {
      "name": "基础条件",
      "start_index": 46,
      "end_index": 80,
      "description": "关于基础设施和条件保障的评估内容",
      "estimated_question_count": 30
    }
  ]
}
```

**维度分析指导**：
- 如果内容没有明显的维度分割，创建一个包含全部内容的单一维度
- 确保维度名称简洁明了，去掉编号前缀
- 维度边界应该是自然的语义分割点
- 估算每个维度可能包含的题目数量
